A Evolução: Do PoC ao Sistema Completo
ANTES — Versão 1.0
  • Senior cria PDF manualmente
  • PDF é ingerido na IA
  • Equipe conversa com a IA
  • Atualizar = recriar o PDF inteiro
  • Sem controle de acesso
  • Dados em texto puro
  • Sem estrutura organizacional
➜
AGORA — Versão 2.0
  • Usuários cadastram conteúdo no sistema
  • Banco: Setor → Área → Tema → Assunto → Mensagens
  • IA carrega sob demanda ao selecionar assunto
  • Atualizar = editar mensagem no sistema
  • Controle de acesso por tipo de usuário
  • Mensagens criptografadas AES-256-GCM
  • Máquina IA 100% offline (sem internet)

⚡ Fluxo de uma Mensagem
1. Usuário salva mensagem
Node.js recebe o texto em claro
2. Criptografia AES-256-GCM
iv:authTag:dadoCifrado → salvo no MySQL
3. Dados ficam no servidor
Nenhum envio automático para a máquina IA
4. IA busca ao selecionar assunto
GET /api/assuntos/:id/mensagens-cifradas
✅ Sob demanda: as mensagens só chegam à máquina IA quando o usuário clicar em "Conversar". Nenhuma sincronização automática.

🔑 Onde a chave fica
Variável de ambiente (.env)
Nunca no banco de dados
Chave: 64 hex chars = 32 bytes
🎲 IV único por mensagem
Vetor gerado aleatoriamente a cada escrita. Nunca reutilizado. Formato: iv:authTag:dado (base64)
✅ Verificação de integridade
GCM detecta qualquer adulteração nos dados cifrados. Dados corrompidos são rejeitados automaticamente.
