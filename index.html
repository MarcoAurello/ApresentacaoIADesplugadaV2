O Problema

A Hemobrás, como qualquer organização de alta complexidade técnica, enfrenta um desafio silencioso: o conhecimento crítico está concentrado em pessoas, não em sistemas. Quando essas pessoas saem, o conhecimento vai com elas.

👴
01
Aposentadorias & Rotatividade
Funcionários experientes acumulam anos de expertise em processos únicos da organização. Quando se aposentam ou são transferidos, levam consigo um acervo de conhecimento que nenhum manual formal conseguiu registrar. A curva de reposição é longa e custosa.
⚠️
02
Conhecimento Tácito Perdido
Procedimentos operacionais, atalhos de sistema, decisões históricas e soluções para problemas recorrentes existem apenas na memória dos veteranos. Esse conhecimento tácito raramente é documentado e quase nunca é transferido de forma estruturada para os mais novos.
🔍
03
Novos Colaboradores sem Referência
Profissionais recém-chegados perdem horas buscando respostas que já foram descobertas e resolvidas internamente. Sem uma base de consulta confiável, recorrem a colegas sobrecarregados, reduzindo a produtividade geral da equipe.

A Evolução: Do PoC ao Sistema Completo

A versão 1.0 validou a viabilidade técnica com um PoC baseado em PDF. A versão 2.0 escala a solução para um sistema robusto, com gestão estruturada, segurança de dados e independência de internet.

ANTES — Versão 1.0 (PoC validado)
Prova de conceito funcional, mas de operação manual e sem estrutura de gestão.
  • Senior escreve um PDF com seu conhecimento
  • PDF é carregado manualmente na IA
  • Equipe conversa com a IA via chat
  • Para atualizar: recriar e reupar o PDF inteiro
  • Sem controle de quem acessa o quê
  • Conteúdo em texto puro no servidor
  • Sem hierarquia organizacional de temas
➜
AGORA — Versão 2.0 (Sistema Completo)
Plataforma com gestão de conteúdo, criptografia e IA desplugada da internet.
  • Usuários cadastram conteúdo diretamente no sistema
  • Hierarquia: Setor → Área → Tema → Assunto → Mensagens
  • IA carrega o conhecimento sob demanda, por assunto
  • Atualizar = editar uma mensagem no sistema
  • Controle de acesso por tipo de usuário
  • Todo conteúdo criptografado com AES-256-GCM
  • Máquina IA totalmente offline — zero dependência de internet

Arquitetura de Conhecimento Estruturada

O conhecimento é organizado em cinco níveis hierárquicos, permitindo que cada área da organização gerencie seu próprio acervo de forma independente e com granularidade total. A IA nunca "mistura" conteúdo de assuntos distintos.

🏢 SetorUnidade organizacional de alto nível — ex: GTIC, GCQ, GPC, GPA...
📂 ÁreaSubdivisão do setor — ex: Infraestrutura, Desenvolvimento, Controle de Qualidade...
📁 TemaAgrupamento de assuntos relacionados — ex: Servidores, Redes, Sistemas, Processos Internos...
📌 AssuntoTópico específico e consultável — ex: Configuração de Firewall, Controle de Ponto, Onboarding...
💬 MensagensConteúdo real cadastrado pelos especialistas — base que alimenta a IA (armazenado criptografado)
💡 Isolamento por sessão: ao selecionar um assunto na IA, somente as mensagens daquele assunto específico são carregadas na sessão de chat. O modelo não tem acesso a nenhum outro conteúdo do sistema durante a conversa.

⚡ Ciclo de Vida de uma Mensagem
Do cadastro pelo especialista até chegar na IA, cada mensagem percorre um pipeline seguro e controlado.
1. Especialista digita o conteúdo
Texto claro submetido via interface de cadastro
2. Criptografia automática no servidor
AES-256-GCM gera: iv:authTag:dadoCifrado (base64) → salvo no MySQL
3. Dados permanecem no servidor interno
Nenhum envio automático para a máquina IA — conteúdo só sai quando solicitado
4. IA busca ao abrir conversa
GET /api/assuntos/:id/mensagens-cifradas — transferência interna na rede local
✅ Controle total: o conteúdo só sai do servidor quando o usuário inicia uma conversa. Não há sincronização automática ou exposição desnecessária dos dados.

Como Funciona — Fluxo da IA

O fluxo foi desenhado para garantir privacidade máxima: as mensagens cifradas viajam da rede interna para a máquina local, onde são decifradas e processadas sem jamais sair da infraestrutura da Hemobrás.

1
🖱️
Seleciona Assunto
O usuário navega pela hierarquia (Setor → Área → Tema) e escolhe o assunto desejado. Em seguida, clica em "Conversar" para iniciar a sessão.
2
📦
Transferência Segura
O backend busca todas as mensagens cifradas do assunto no MySQL e as envia via rede interna para a máquina IA local. Nenhum dado trafega pela internet.
3
🧠
Ingest Local
A máquina IA decifra as mensagens, gera os embeddings vetoriais e indexa no ChromaDB local. Se o mesmo assunto já estiver carregado, esta etapa é pulada automaticamente.
4
💬
Chat Inteligente
Busca híbrida BM25 + Vetorial captura tanto termos exatos quanto intenção semântica. Reranker seleciona os melhores chunks. LLM Llama responde com temperature=0 para máxima precisão.
🔒 Máquina IA sem internet  ·  ChromaDB persiste localmente  ·  "🔄 Trocar de Assunto" limpa a sessão e recarrega novo contexto

Segurança de Dados
🔐 Criptografia AES-256-GCM  ·  Padrão Militar  ·  Dados ilegíveis no banco sem a chave secreta

O conteúdo das mensagens é tratado como dado sensível desde o momento do cadastro. Mesmo que alguém acesse diretamente o banco de dados MySQL, verá apenas sequências cifradas incompreensíveis sem a chave correta.

🔑 Gestão da Chave
A chave de 256 bits vive exclusivamente em variável de ambiente (.env). Nunca é armazenada no banco de dados. Apenas os servidores autorizados têm acesso a ela.
🎲 IV Único por Mensagem
Cada mensagem recebe um vetor de inicialização (IV) gerado aleatoriamente na escrita. Mesmo conteúdos idênticos produzem ciphertext completamente diferente — impossível de correlacionar.
✅ Integridade Garantida
O modo GCM inclui uma tag de autenticação (AuthTag) que detecta qualquer adulteração nos dados cifrados. Modificações maliciosas são automaticamente rejeitadas antes da decriptação.

Rotas da API

O sistema é dividido em dois serviços independentes: o backend de gestão (Node.js na rede interna) e o serviço de IA (Python na máquina local sem internet). A comunicação entre eles é pontual e sempre com dados cifrados.

🟢 Backend de Gestão — Node.js (servidor interno)
Gerencia usuários, hierarquia e mensagens. Toda escrita no banco passa pela camada de criptografia antes de ser persistida.

CRUD/api/tipos-usuario / setores / areas
CRUD/api/usuarios / temas / assuntos

PUT/api/mensagens/:idcifra + atualiza

🔵 Serviço IA — Python FastAPI (máquina local, sem internet)
Recebe dados cifrados da rede interna, processa localmente e responde ao usuário. Nenhuma chamada externa é feita.

Próximos Passos

Com a arquitetura definida e o PoC aprovado, a implementação segue uma sequência lógica: banco de dados → backend → serviço IA → interface. Cada etapa entrega valor independente e pode ser validada antes da próxima.

01
Banco de Dados (MySQL + Sequelize)Criar e migrar as tabelas: TipoUsuario, Setor, Area, Usuario, Tema, Assunto e Mensagens. Configurar as chaves estrangeiras e o modelo de dados completo conforme arquitetura definida.
02
Backend de Gestão (Node.js + Express)Desenvolver o CRUD completo para toda a hierarquia, incluindo autenticação JWT, módulo de criptografia AES-256-GCM e a rota GET /assuntos/:id/mensagens-cifradas para servir a máquina IA.
03
Serviço IA (Python FastAPI — máquina local)Adaptar o rag-poc existente: implementar as rotas /ingest (decifra + ChromaDB), /query (busca híbrida + LLM) e /limpar (troca de assunto). Substituir filtro de source pelo assunto_id.
04
Interface (Next.js) + PilotoConstruir a aba de cadastro protegida por senha, o seletor de assunto com hierarquia navegável, o chat com botão "🔄 Trocar de Assunto" e lançar o piloto em um setor para coleta de feedback.
